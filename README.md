# Gemini CLI Toolset

This project provides a suite of secure tools for a Large Language Model (specifically Google Gemini) to interact with a local file system and execute Python code.

## Features

- **File Information (`get_files_info`)**: Lists files in a specified directory, providing file sizes and identifying directories.
- **Read File Content (`get_file_content`)**: Retrieves the text content of files, with automatic truncation to prevent context window overflow.
- **Write File (`write_file`)**: Writes content to new or existing files, automatically creating subdirectories as needed.
- **Run Python File (`run_python_file`)**: Securely executes Python scripts locally and returns the execution output (STDOUT/STDERR).

## Security

Security is a primary design consideration:
- **Path Validation**: All tools use `os.path.commonpath` and `os.path.normpath` to ensure that operations cannot escape the permitted `working_directory`.
- **Directory Traversal Protection**: Prevents access to system directories or parent directories (e.g., `/bin`, `../`).
- **Execution Isolation**: Python execution is performed via subprocesses with defined timeouts (30 seconds).

## Project Structure

- `functions/`: Contains the implementation of the file system and execution tools.
- `functions/call_function.py`: The integration hub. Defines `genai.types.FunctionDeclaration` schemas and handles the mapping of model tool calls to Python functions.
- `config.py`: Contains global configuration settings such as `MAX_CHARS` for output truncation.

## Setup

1. **Configuration**:
   Ensure you have a `config.py` defining `MAX_CHARS`.
   
2. **Environment**:
   Create a `.env` file in the root directory:
   ```env
   GOOGLE_API_KEY=your_api_key_here
   ```

3. **Install Dependencies**:
   ```bash
   pip install google-genai python-dotenv
   ```

## Usage

The `call_function` module is used to process tool calls generated by the Gemini model. By default, the orchestrator is configured to restrict operations to the `./calculator` directory.

```python
from functions.call_function import call_function

# Example dispatch of a function call from the model
# response = call_function(model_generated_function_call)
```

All functions return detailed strings or error messages starting with `Error:` to provide immediate feedback to the model.
